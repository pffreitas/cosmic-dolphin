## 2024-06-11 - Implement SSRF Protection in Web Scraper
**Vulnerability:** The application fetches external content (web scraping, link previews) via `got` without validating the hostnames. This allows Server-Side Request Forgery (SSRF) vulnerabilities where an attacker could provide URLs like `http://localhost:3000` or `http://169.254.169.254` (cloud metadata service), potentially reading sensitive internal endpoints or traversing the internal network.
**Learning:** Static string checks (e.g. `hostname === 'localhost'`) are insufficient because attackers can bypass them using DNS (e.g. `localtest.me`), decimal IPs, IPv6 mapping, or redirection. A robust solution must fail closed, intercept the request immediately before it's sent via the `beforeRequest` hook in `got`, perform a DNS lookup, and explicitly block private and local IP ranges (like `127.0.0.0/8`, `10.0.0.0/8`, etc.), then route the request directly to the validated IP while preserving the original `Host` and SNI.
**Prevention:** Always use DNS resolution to get the underlying IP address before making outbound requests to user-supplied URLs. Validate the IP against a comprehensive blocklist of internal networks. Use `got`'s `beforeRequest` hook to enforce this check transparently. Set `https.servername` to ensure SNI validation does not fail when replacing hostname with IP.

## 2024-06-12 - Handle IPv6 and Local Blocklists properly
**Vulnerability:** Node.js native URL logic has bugs/features that allow IPv6-based DNS rebinding or direct access to bypass traditional SSRF blocklists. `URL.hostname` for IPv6 fails silently when assigned to a non-bracketed IPv6 IP. The blocklist was also missing `172.16.0.0/12` ranges and IPv6 internal spaces.
**Learning:** `got`'s beforeRequest hook SSRF protection needs to explicitly check `net.isIPv6(ip)` and wrap it in brackets `[${ip}]` before assigning it to `options.url.hostname`, or else `got` will re-resolve the original hostname. We also need to add IPv6 local/private spaces (`fc`, `fd`, `fe8`, etc.) and the missing `172.16.*`-`172.31.*` ranges.
**Prevention:** Use a comprehensive array of blocked IP prefixes for both IPv4 and IPv6, and always use explicit IPv6 formatting when overriding URL object properties.
