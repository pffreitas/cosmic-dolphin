## 2024-05-18 - Missing Startup Secret Validation
**Vulnerability:** The API configuration (`apps/api/src/config/environment.ts`) used unvalidated `process.env` values for critical secrets like `DATABASE_URL`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, and `JWT_SECRET`, falling back to empty strings or default weak values when not present.
**Learning:** This architectural flaw allowed the service to start even if critical credentials were missing or incorrectly formatted. When secrets are empty strings, subsequent authentication logic or database connections can behave unpredictably, leading to either immediate, cryptic errors deep in the application or unauthorized access if default values are unintentionally exploited.
**Prevention:** Enforce strict validation at startup. Use `zod` to validate environment configurations right where they are loaded. Ensure critical secrets are required (`min(1)`), URLs are validated, and keys like `JWT_SECRET` meet a minimum complexity length (e.g. 32 characters) so the application fails fast and securely rather than operating in a compromised state.
